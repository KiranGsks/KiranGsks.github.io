**Examples:** Pokémon GO Watch features

## 13. TECHNICAL CONSTRAINTS & DESIGN PHILOSOPHY

### Wear OS Platform Limits

| Constraint | Impact on Design |
|------------|------------------|
| **Small Screen** | 1.2-1.8" circular/square - minimalist UI required |
| **Limited CPU/GPU** | 2D graphics preferred, optimize rendering |
| **Battery Drain** | Short sessions critical (<2% per 10 min) |
| **Input Model** | Gyro/accelerometer/haptic > touch |
| **Network** | Intermittent connectivity - offline-first design |

### Design Principles

1. **Haptics as UI**
   - Distinct patterns for lock/danger/combo/near-miss
   - Reduces text clutter on tiny screens
   - Works with gloves/wet hands

2. **Short Sessions**
   - 15-60 second rounds
   - Minimizes screen-on time
   - Perfect for commute/waiting scenarios

3. **Gyro-First Mechanics**
   - Tilt = primary input
   - Tap = secondary action
   - Crown button = menu/pause

4. **Procedural Content**
   - Template-based levels
   - Infinite scalability without asset bloat
   - Low storage footprint

5. **Hybrid-Casual Meta**
   - Simple core loop
   - Light progression (streaks, dailies)
   - Cosmetic upgrades (non-pay-to-win)

### Performance Targets

| Metric | Target | RetroTennis Current |
|--------|--------|---------------------|
| App Size | <5MB | 4.2MB |
| Load Time | <2 seconds | 1.8s |
| Frame Rate | 30 FPS min | 30 FPS |
| Battery Drain | <2% per 10 min | 1.7% |
